# P2Line 线稿提取工具

## 主要功能

XDOG 线稿提取：
  - 基于高斯差分算法，提供精细的边缘检测控制。
  - 参数调节：支持 Sigma（模糊半径）、K（模糊倍率）、Tau（边缘强度）、Phi（软阈值）、Epsilon（阈值偏移）等参数微调。
  - 后处理：支持去噪点（去除孤立像素）、断线连接（闭合微小缺口）、线条加粗。

PS 风格线稿提取:
  - 模拟 Photoshop 的线稿提取工作流：去色 -> 反相 -> 颜色减淡 -> 最小值滤镜 -> 色阶调整。
  - 参数调节：
  - 最小值半径：控制线条的粗细和细节保留程度。
  - 色阶调整：通过黑白场和 Gamma 值调整线稿的对比度和清晰度。

### 点集提取
专为 hexo博客首页动画设计的点集生成功能，支持三种提取模式：
  - 顺序绘制：  按网格扫描图片，生成规则的点阵。
  - 路径化绘制：沿线条路径追踪，生成连续的线条点集。      **（算法待优化，当前版本生成效果可能欠佳）**
  - 轮廓绘制：  提取图像的最外层轮廓，适合生成物体边缘。  **（算法待优化，当前版本生成效果可能欠佳）**

### 通用功能
图片裁剪：支持自定义裁剪区域，仅提取感兴趣部分的线稿。
颜色控制：支持反色（黑底白线/白底黑线）和透明度调整。
多格式导出：
  - 图片格式：PNG, JPG
  - 矢量格式：SVG (支持路径优化)
  - 数据格式：JSON (包含点集坐标，用于二次开发)

## 快速开始

### 本地开发

**克隆项目**
```bash
git clone https://github.com/your-username/p2line.git
cd p2line
```

**安装依赖**
```bash
npm install
```

**启动开发服务器**
```bash
npm run dev
```

**构建生产版本** (不建议本地部署，可以考虑暂时使用dev环境)
```bash
npm run build
```

## 点集数据格式

当导出格式选择 **点集 JSON** 时，生成的 JSON 文件包含一个坐标数组，格式如下：

```json
[
  { "x": 10, "y": 20 },
  { "x": 11, "y": 21 },
  ...
]
```


## Vercel 部署教程

本项目基于 Vue 3 + Vite 构建，有需要可以部署在 Vercel 上。

如果你选择fork仓库，当代码有更新时，你需要手动进行同步，才能在 Vercel 上部署最新版本。

或者你可以考虑直接使用原仓库的url进行部署：[https://github.com/luoy-oss/p2line](https://github.com/luoy-oss/p2line)



原仓库部署流程：

> 登录 Vercel 账号。
>
> 点击 **"Add New..."** -> **"Project"**。
>
> 在新建项目页面上方的url输入框中输入原仓库地址：https://github.com/luoy-oss/p2line
>
> 点击右侧出现的Deploy
>
> 相关配置项默认即可。
>
> 点击 **"Deploy"** 按钮完成部署。
>
> 等待几分钟，构建完成后，Vercel 会提供一个访问链接（例如 `https://your-project.vercel.app`）
>
> 点击链接即可访问部署后的 P2Line 线稿提取工具。



fork部署流程：

> 准备工作
>
>   - fork 本项目到你的 GitHub 仓库。
>   - 注册并登录 [Vercel](https://vercel.com)。
>
> 导入项目
>
>   - 在 Vercel 仪表盘点击 **"Add New..."** -> **"Project"**。
>   - 选择你fork的 Git 仓库并点击 **"Import"**。
>
> 配置项目
>
>   - 默认即可。
>
> 部署
>
>   - 点击 **"Deploy"** 按钮。
>   - 等待几分钟，构建完成后，Vercel 会提供一个访问链接（例如 `https://your-project.vercel.app`）
>   - 点击链接即可访问部署后的 P2Line 线稿提取工具。
